## ✏️ 개별 문제 정리 방식

각 문제에 대한 자세한 정리는 다음 두 가지 방식 중 택일 혹은 병행합니다.

1. **이 README 안에 섹션으로 정리**

   예)

   ### [BOJ 1000 - A+B](https://www.acmicpc.net/problem/1000)

   - 난이도: Bronze V
   - 유형: 구현
   - 언어: JavaScript (Node.js)

   - 핵심 아이디어
     - 입력으로 들어오는 두 정수를 공백 기준으로 split
     - Number로 변환 후 덧셈

   - 시간 복잡도: O(1)

2. **개별 노트 파일 사용**

   - `notes/` 아래에 **유형별 정리**를 해 놓고,
     각 문제는 “관련 개념” 섹션에 링크를 추가합니다.
   - 예: `notes/05_two_pointer_prefixsum/two_pointer.md` 안에
     “연습 문제: [BOJ 2003 - 수들의 합 2] 링크” 추가

> 실제 사용하는 언어/환경에 맞게 수정해서 사용합니다.

  - 로컬에서 테스트 시

- Python

- 티어별로 최소 목표

  - Bronze: **기본 문법 & 구현 감각 익히기**
  - Silver: **코딩 테스트 기본기 완성**

    하는 흐름을 만드는 것을 목표로 합니다.
