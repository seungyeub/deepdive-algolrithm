# 📝 재귀함수 (Recursion)

> 함수가 자기 자신을 다시 호출하여 문제를 해결하는 기법입니다. "큰 문제를 동일한 구조의 작은 문제로 쪼갠다"는 것이 핵심입니다.

## 📚 개념 요약
- **재귀 호출(Recursive Case):** 문제를 더 작은 단위로 쪼개어 자기 자신을 다시 호출하는 부분.
- **기초 조건(Base Case):** 재귀를 멈추는 조건. 이 조건이 없으면 무한 루프에 빠져 **Stack Overflow**가 발생합니다.

## 💡 핵심 아이디어 / 자주 쓰는 패턴
- **하향식 사고:** "10까지의 합은 (10 + 9까지의 합)이다"와 같이 결과 중심적으로 생각하기.
- **스택 구조:** 함수가 호출될 때마다 메모리의 스택(Stack) 영역에 쌓이며, 가장 나중에 호출된 함수가 먼저 종료되는 LIFO 방식입니다.

## ⚠️ 자주 나오는 실수 / 주의사항
- **Base Case 누락:** 종료 조건이 없거나 도달할 수 없는 조건일 경우 프로그램이 다운됩니다.
- **성능 이슈:** 반복문(for, while)보다 메모리 사용량이 많고 속도가 느릴 수 있습니다. (함수 호출 오버헤드 때문)
- **깊은 재귀:** 호출 스택이 너무 깊어지면 JS 엔진의 제한(Maximum call stack size exceeded)에 걸릴 수 있습니다.
