# ⚖️ 정렬(Sorting) vs 빈도수(Frequency Map)

## 1. 정렬 방식 (Sorting)
- **전략:** 데이터를 순서대로 나열하여 특정 위치(인덱스)의 값을 비교.
- **장점:** 코드가 매우 직관적이고 눈으로 검증하기 쉬움.
- **적용:** 주사위 게임처럼 데이터 양이 적고 인접한 값의 관계가 중요할 때.

## 2. 빈도수 방식 (Frequency Map)
- **전략:** `{ 값: 등장횟수 }` 형태의 객체를 만들어 통계 데이터를 추출.
- **장점:** 데이터가 섞여 있어도 상관없으며, 데이터 양이 많아져도 코드가 길어지지 않음.
- **적용:** 어떤 항목이 가장 많이 나왔는지(최빈값) 찾아야 할 때.



### 💡 선택 기준
- "눈에 보이는 순서가 중요한가?" → **정렬**
- "무엇이 몇 번 나왔는지가 중요한가?" → **빈도수**
