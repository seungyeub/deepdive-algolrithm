# 💡 상태 관리와 토글 (State Management & Toggle)

> 알고리즘 문제에서 특정 조건에 따라 프로그램의 동작 방식이나 규칙이 실시간으로 변해야 할 때 사용하는 패턴입니다.

## 🔑 상태(State)의 개념
- **정의:** 프로그램의 현재 '모드'나 '상황'을 기억하는 변수입니다.
- **역할:** 같은 입력이 들어와도 현재 상태(`mode`)에 따라 다른 로직을 실행하게 합니다.
- **예시:** `mode 0` (짝수 인덱스 수집) vs `mode 1` (홀수 인덱스 수집)

## 🔄 주요 패턴: 토글 (Toggling)
두 가지 상태를 번갈아 가며 전환해야 할 때 사용하는 효율적인 코드 패턴들입니다.

### 1. 산술 연산 활용 (가장 간결 ⭐)
정수 0과 1을 스위칭할 때 매우 유용합니다.
```javascript
mode = 1 - mode; // 0 -> 1, 1 -> 0
```

### 2. 비트 연산 활용 (심화 테크닉)
XOR(^) 연산은 비트가 서로 다를 때 1을 반환하는 특성을 이용합니다.
```javascript
mode ^= 1; // 0과 1을 가장 빠르게 스위칭
```

### 3. 논리 연산 활용
불린(Boolean) 값을 다룰 때 직관적입니다.
```javascript
isModeOne = !isModeOne;
```

## 🧩 시뮬레이션 구현 전략
문제를 풀 때 로직을 **[상태 변경]**과 **[데이터 처리]**로 명확히 분리하면 실수가 줄어듭니다.

1.  **상태 정의:** 동작을 결정하는 변수 설정 (예: `mode = 0`)
2.  **변화 조건(Trigger):** 상태가 바뀌는 시점 파악 (예: 문자 `"1"`을 만났을 때)
3.  **분기 구현:** 각 상태에서 수행할 로직을 독립적으로 작성
    - `if (mode === 0) { ... }`
    - `else if (mode === 1) { ... }`



## ⚠️ 자주 하는 실수 (Troubleshooting)
- **업데이트 순서:** "모드를 바꾼 후 문자를 추가할 것인가?" 아니면 "추가한 후 모드를 바꿀 것인가?"에 따라 결과가 완전히 달라집니다. (문제의 요구사항을 정밀하게 읽어야 함)
- **초기값 미설정:** 시작 시점의 `mode`가 0인지 1인지에 따라 결과가 반전될 수 있습니다.
