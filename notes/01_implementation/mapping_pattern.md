## ⚖️ 구현 방식별 비교 분석 (수 조작하기 2 사례)

| 방식 | 가독성 | 메모리 효율 | 특징 |
| :--- | :---: | :---: | :--- |
| **Mapping + for** | 상 | 상 | 추천하는 균형 잡힌 방식 |
| **slice + map** | 최상 | 하 | 간결하지만 새로운 배열 생성 비용 발생 |
| **if-else + for** | 하 | 상 | 가장 빠르지만 코드가 장황함 |

### 💡 인사이트
알고리즘 문제의 제한 사항에서 **배열의 크기가 크다면(예: 10^5 이상)** 복사본을 만드는 `slice`, `filter`, `map` 체이닝보다는 **원본 배열에 인덱스로 접근하는 `for` 루프**를 사용하는 것이 안전합니다.
